## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### System Cleaning

- [x] **SYS-01**: User can scan and remove user app caches from ~/Library/Caches
- [x] **SYS-02**: User can scan and remove user app logs from ~/Library/Logs
- [x] **SYS-03**: User can scan and remove QuickLook thumbnail cache

### Browser Data

- [x] **BRWS-01**: User can scan and remove Safari cache
- [x] **BRWS-02**: User can scan and remove Chrome cache
- [x] **BRWS-03**: User can scan and remove Firefox cache

### Developer Caches

- [x] **DEV-01**: User can scan and remove Xcode DerivedData
- [x] **DEV-02**: User can scan and remove npm/yarn cache
- [x] **DEV-03**: User can scan and remove Homebrew cache
- [x] **DEV-04**: User can scan and remove Docker unused images/containers/volumes

### App Leftovers

- [x] **APP-01**: User can scan and remove orphaned preferences from uninstalled apps
- [x] **APP-02**: User can scan and remove old iOS device backups
- [x] **APP-03**: User can scan and remove old files from ~/Downloads (by age)

### CLI Interface

- [x] **CLI-01**: User can preview cleanup with `--dry-run` (no files deleted)
- [x] **CLI-02**: User can target all categories with `--all`
- [x] **CLI-03**: User can run interactive mode (no args) that walks through each item
- [x] **CLI-04**: User sees confirmation prompt before any deletion
- [x] **CLI-05**: User sees summary after cleanup (items removed, space freed)
- [x] **CLI-06**: User can get structured JSON output with `--json`
- [x] **CLI-07**: User can get detailed file listing with `--verbose`
- [x] **CLI-08**: User can skip categories with `--skip-<category>` flags
- [x] **CLI-09**: User can skip specific items with `--skip-<item>` flags
- [x] **CLI-10**: User can bypass confirmation with `--force` for automation

### Safety

- [x] **SAFE-01**: Tool never touches SIP-protected paths (/System, /usr, /bin, /sbin)
- [x] **SAFE-02**: Tool never touches swap files or sleepimage
- [x] **SAFE-03**: Tool reports what it can't access (permission issues) without failing
- [x] **SAFE-04**: Tool categorizes items by risk level (safe/moderate/risky)
